# Session Flow

> 아침 5분 발산 세션과 저녁/주말 수렴 세션의 운영 흐름과 리듬을 정의한다.

---

## 운영 원칙

- 아침에는 ==빠른 발산만== 수행한다.
- 저녁이나 주말에는 선택한 아이디어만 깊게 검토한다.
- 전체 워크플로우를 한 세션에 몰아넣지 않는다.

> [!tip]
> 발산과 수렴을 분리하면 의사결정 피로를 줄이고 아이디어 품질을 높일 수 있다.

---

## 아침 5분 발산 세션 (매일)

```text
[0:00] 앱 열기 → 오늘의 추천 조합 3세트 표시
[0:30] 추천 조합 수락 또는 직접 키워드 2~3개 선택
[1:00] 생성 모드 선택 (Full Match / Forced Pairing)
[1:30] LLM이 아이디어 10개 생성
[3:30] 마음에 드는 1~2개 북마크
[4:00] Firestore에 자동 저장 (북마크 + 비선택 모두 기록)
[4:30] 종료
```

### 운영 Fallback

- 발산 세션이 ==주 3회 이상 5분을 넘기면==, 다음 주부터 상위 3개 후보 + 나머지 펼치기 방식으로 UI 축소
- 북마크 선택이 반복해서 지연되면 요약 길이를 더 짧게 제한

> [!warning]
> 발산 세션이 길어지면 습관 형성에 방해가 된다. 5분 제한을 엄격히 유지할 것.

---

## 저녁/주말 수렴 세션 (선택적)

```text
1. 북마크된 아이디어 중 1개 선택
2. Deep Report(PRD) 생성
3. 비즈니스 평가 매트릭스 실행
4. Next Step 확인
5. 상세 페이지 저장
6. 아이디어 상태 업데이트
```

---

## 운영 리듬

| 주기 | 운영 내용 |
|---|---|
| 매일 | 발산 세션 1회 수행, 필요 시 커스텀 키워드 추가 |
| 주 1~2회 | Deep Report 생성 및 평가 진행 |
| 주간 | 북마크 아이디어 검토, 상태 전환, 보류 여부 결정 |
| 분기 1회 | 전체 키워드 풀 리뷰 및 정리 |

추가 원칙:
- 추가 발산 세션이 필요하면 허용하되 ==비용 경고를 표시==한다.
- ==14일 이상 상태가 멈춘 아이디어==는 재검토 대상으로 올린다.
