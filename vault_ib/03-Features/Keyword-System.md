# Keyword System

> 아이디어 생성의 핵심 축인 ==5개 카테고리 키워드 풀==의 구조, 운영 정책, 정규화 규칙을 정의한다.

---

## 키워드 카테고리

Idea Bank의 아이디어 생성은 ==5개 축의 키워드 조합==을 중심으로 돌아간다.

| # | 카테고리 | 설명 |
|---|----------|------|
| 1 | **Who** | 타겟 사용자층 |
| 2 | **Domain** | 산업/분야 |
| 3 | **Tech** | 핵심 기술 |
| 4 | **Value** | 사용자 체감 가치 |
| 5 | **Money** | 수익화 메커니즘 |

## 고정 키워드 풀 (Fixed Pillars)

> [!note]
> 전체 키워드 원본은 [[Keyword-Pool]] 참조.

| 카테고리 | 개수 | 예시 |
|----------|------|------|
| Who | 20 | 1인 가구, Z세대, 디지털 노마드, 소상공인 |
| Domain | 22 | 핀테크, 헬스케어, 에듀테크, 멘탈헬스 |
| Tech | 17 | LLM 에이전트, RAG, 멀티모달, 노코드 |
| Value | 18 | 시간 단축, 생산성, 비용 절감, 연결성 |
| Money | 19 | 구독형, 프리미엄, 중개 수수료, API 과금 |

## 커스텀 키워드

사용자가 직접 추가/삭제하는 키워드.

- V1에서는 웹 UI를 통해 ==수동 관리==
- 키워드 데이터 구조는 고정 키워드와 동일 (`source: "custom"`)
- V2 다이내믹 키워드와 연결할 수 있도록 구조 통일

## 다이내믹 키워드 (V2)

> [!warning]
> V1 안정화 이전에는 도입하지 않는다.

- 최신 IT 뉴스와 SNS 트렌드를 분석해 ==신규 키워드를 자동 생성==
- 뉴스 소스 선정, 수집 파이프라인, 키워드 추출, 중복 제거, 스케줄링 필요

## 추상도 정규화 규칙

카테고리별 추상도는 아래 수준을 기본으로 맞춘다.

| 카테고리 | 기준 추상도 |
|----------|-------------|
| Who | 사용자군 또는 생활/업무 맥락 수준 |
| Domain | 산업 vertical 수준 |
| Tech | 구현 접근 방식 또는 핵심 기술 수준 |
| Value | 사용자가 체감하는 효용 수준 |
| Money | 수익화 메커니즘 수준 |

> [!tip]
> 분기 리뷰 때 너무 넓거나 좁은 항목은 ==sibling 수준으로 재작성==하거나 분리한다.

## 키워드 풀 운영 정책

| 주기 | 액션 |
|------|------|
| 매일 | 커스텀 키워드 수동 추가 가능 |
| 분기 1회 | 전체 키워드 풀 리뷰 |
| 리뷰 기준 | 의미적 중복 여부, 추상도 균형, 3개월 미사용 키워드, 신규 트렌드 반영 |
