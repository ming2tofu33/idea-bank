# Evaluation Matrix

> 아이디어를 ==4개 카테고리 가중 점수==로 평가하고, 3중 출력 구조로 hallucination을 완화하는 매트릭스를 정의한다.

---

## 평가 항목과 가중치

| 카테고리 | 가중치 | 평가 기준 |
|----------|--------|-----------|
| **시장성 (Market)** | ==0.30== | Problem-Solution Fit, 시장 규모, 타겟 명확성 |
| **실행 가능성 (Build)** | ==0.25== | 기술 난이도, MVP 제작 소요 기간 |
| **독창성 (Edge)** | ==0.25== | 차별성, 의외성, 경쟁 우위 가능성 |
| **수익성 (Money)** | ==0.20== | 수익 모델의 명확성, 지속 가능성 |

> [!warning]
> **윤리성(Ethics):** 별도 플래그로 관리. 점수에 포함하지 않되, ==치명적 윤리 리스크 발견 시 경고 플래그==를 부여한다.

## 3중 출력 구조 (Hallucination 완화)

> [!important]
> 각 평가 항목은 반드시 아래 3중 포맷으로 출력한다.

```
시장성: 72/100
━━━━━━━━━━━━━━━━━━━━━━
근거: [왜 이 점수인지 구체적 이유 2~3문장]
반론: [이 점수가 과대/과소 평가일 수 있는 이유 1~2문장]
확인 필요: [검증하면 점수가 바뀔 수 있는 핵심 가정 1개]
```

| 요소 | 역할 |
|------|------|
| **점수** | 절대값보다 ==상대 비교용 수치==로 사용 |
| **근거** | 가장 중요한 아웃풋. 점수의 이유 |
| **반론** | 자기 의심을 구조화해 ==hallucination 가시화== |
| **확인 필요** | 후속 검증 체크리스트 역할 |

> [!tip]
> 기준선: ==60점이 평균==. 점수 인플레이션 방지.

## Next Step 규칙

| 종합 점수 | 제안 액션 |
|-----------|-----------|
| **80+** | 경쟁사 3곳 심층 리서치, 랜딩 페이지 초안 작성, 1주 MVP 스프린트 계획 |
| **60~79** | 확인 필요 항목 우선 리서치, 피벗 포인트 탐색 |
| **60 미만** | 핵심 가치 재정의, 키워드 조합 변경, 보류 처리 |

## 평가 후 상태 전이

| 조건 | 상태 전이 |
|------|-----------|
| 종합 점수 80 이상 | `reviewing` 유지, ==실행 후보 태그== 부여 |
| 종합 점수 60~79 | `interested` 또는 `reviewing` 유지, 추가 리서치 필요 태그 |
| 종합 점수 60 미만 | 기본값 `on_hold` |
| 중복 경고 발생 | ==저장하되 경고 표시==, 사용자가 유지/삭제 판단 |
| 보류 아이디어 재검토 | 수동 승인 시 `interested`로 복귀 |
| 14일 방치 | → `archived` (자동) |
