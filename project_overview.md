# 🚀 아이디어 팩토리 (Idea Factory: Vibe Workbench)

> **"매일 아침 5분, 체계적 키워드 조합으로 비즈니스 아이디어를 발산하고 축적하는 개인 생산성 엔진"**

---

## 1. 프로젝트 비전 (Vision)

본 프로젝트는 AI 엔지니어가 매일 새로운 비즈니스 영감을 얻을 수 있도록 돕는 인터랙티브 아이디어 워크벤치이다. 고정된 비즈니스 프레임워크 키워드와 사용자 커스텀 키워드를 결합하여, 인간의 직관(Vibe)과 AI의 생성 능력이 시너지를 내는 'Human-in-the-Loop' 시스템을 지향한다.

### 1.1 핵심 차별점

기존 대안(ChatGPT Custom GPTs, Notion AI, IdeaBuddy, Validator AI 등)과의 차이는 다음 세 가지에 있다:

- **체계적 키워드 조합:** 단순 "아이디어 내줘"가 아닌, 5개 카테고리 × 90+ 키워드 매트릭스 기반의 구조적 발산
- **평가 프레임워크 내장:** 생성과 평가가 하나의 파이프라인으로 연결되어, 아이디어를 "재미있다" 수준에서 "실행할 만하다" 수준으로 끌어올림
- **축적과 학습:** 아이디어가 일회성이 아니라 Notion에 구조화되어 쌓이고, 선택/비선택 패턴이 시스템 개선에 활용됨

### 1.2 프로젝트 조건

| 항목 | 결정 |
| --- | --- |
| 타겟 사용자 | 본인 (AI 엔지니어, 개인 생산성 도구) |
| 사용 패턴 | 매일 아침 5분 (발산), 저녁/주말 (수렴, 선택적) |
| 저장소 | Notion |
| 기술 스택 | V1: Claude Project + Notion API → V2: Next.js 웹앱 마이그레이션 |
| 월 예상 API 비용 | 상세 내용은 §8 참고 |

## 2. 핵심 키워드 전략 (Keyword Strategy)

### 2.1 고정 키워드 (Fixed Pillars)

| 카테고리 | 키워드 리스트 |
| --- | --- |
| **1. Who (타겟층)** | 1인 가구, Z세대, 알파세대, 실버 세대, N잡러, 딩크족, 주니어 개발자, 반려동물 가구, 디지털 노마드, 소상공인(SME), 취준생, 외국인 거주자, 1인 크리에이터, 환경 운동가, 나홀로 여행객, 사이드 프로젝트 팀, 중소기업 의사결정자(C-level), 워킹맘/워킹대디, 장애인/접근성 사용자, 디지털 네이티브 시니어 |
| **2. Domain (산업)** | 핀테크, 헬스케어, 에듀테크, 커머스, 로지스틱스, 스마트홈, 로컬 커뮤니티, 프롭테크, K-컬처, 뷰티테크, 멘탈헬스, 트래블테크, 리걸테크, HR-tech, F&B(외식업), 엔터테인먼트, 슬립테크, 스페이스테크, 기후테크, 사이버보안, 농업테크(AgriTech), 데이팅/관계 |
| **3. Tech (기술)** | LLM 에이전트, RAG(검색 증강), 멀티모달, 노코드/로코드, 엣지 AI, 웨어러블, 개인화 알고리즘, 자동화 워크플로우, 벡터 데이터베이스, TTS/STT, 이미지 생성 AI, 온디바이스 AI, 추천 엔진, 데이터 시각화, 디지털 트윈, 지식 그래프, 합성 데이터 |
| **4. Value (가치)** | 시간 단축, 외로움 해소, 생산성 극대화, 자아실현, 비용 절감, 심리 케어, 연결성, 재미(Gamification), 안전/보안, 결정 피로 감소, 소속감, 스킬 향상, 데이터 프라이버시, 창의적 영감, 건강 트래킹, 노력의 시각화, 심리적 안전감, 디지털 디톡스 |
| **5. Money (수익)** | 구독형(SaaS), 프리미엄(Freemium), 중개 수수료, 데이터 판매, 광고 기반, API 과금, D2C(직판), 화이트 라벨링, 건당 결제(Pay-per-use), 라이선싱, 리퍼럴(제휴), 하드웨어 결합형, 크라우드 펀딩, 성공 보수형, 티어드 프라이싱, 토큰 경제, 번들링, 마켓플레이스 수수료, 임베디드 파이낸스 |

> **키워드 풀 관리 원칙:**
> - 분기 1회 키워드 풀 리뷰 (중복 제거, 신규 추가, 추상도 균형 점검)
> - 각 카테고리 내 키워드는 유사한 구체성 수준을 유지
> - 의미적 중복 키워드는 하나로 통합 (예: "엣지 AI"와 "온디바이스 AI" → 맥락에 따라 하나 선택 또는 병합)

### 2.2 커스텀 키워드 (사용자 추가)

- 사용자가 직접 키워드를 추가/삭제할 수 있는 기능 제공
- V2의 다이내믹 키워드 자동화로 자연스럽게 확장 가능
- 키워드 데이터 구조에 반드시 아래 필드 포함:

```json
{
  "keyword": "AI 코파일럿",
  "category": "Tech",
  "source": "custom",       // "fixed" | "custom" | "dynamic"(V2)
  "added_date": "2025-03-07",
  "used_count": 0,
  "last_used": null
}
```

### 2.3 다이내믹 키워드 (V2 예정)

- **V1:** 수동 커스텀 키워드로 대체. 위 데이터 구조를 통해 V2 전환 준비
- **V2:** LLM이 최신 IT 뉴스 및 SNS 트렌드를 분석하여 매일 20개의 신규 키워드 자동 생성
  - 뉴스 소스 선정, 크롤링 파이프라인, 키워드 추출/중복 제거 로직, 스케줄러 필요
  - V1 안정화 이후 착수

## 3. 아이디어 생성 모드 (Generation Modes)

### 3.1 Full Match 모드

사용자가 선택한 모든 키워드를 반드시 반영하여 아이디어 생성. 복합적인 비즈니스 설계 시 유리하다.

### 3.2 Forced Pairing 모드

"고정 키워드 2개 + 커스텀 키워드 1개(있을 경우)"를 강제로 조합. 의외의 혁신적인 아이디어 도출 시 유리하다.

### 3.3 오늘의 추천 조합 (Serendipity Mode)

매일 아침의 결정 피로를 줄이기 위한 핵심 기능:

- 이전에 사용하지 않은 키워드 조합을 우선 추천
- 최근 자주 선택한 키워드와 의도적으로 다른 조합 제안
- "추천 조합 수락 → 바로 생성"으로 최단 경로 제공
- 추천 로직: `사용 빈도 낮은 키워드 가중 + 카테고리 간 다양성 확보`

## 4. 워크플로우 (Workflow)

### 4.1 세션 분리 전략

5분 안에 전체 워크플로우를 실행하는 것은 비현실적이므로, 세션을 **발산(아침)**과 **수렴(저녁/주말)**으로 분리한다.

### 4.2 아침 5분 — 발산 세션 (매일)

하루 사용 시나리오:

```
[0:00] 앱 열기 → "오늘의 추천 조합" 3세트 표시
[0:30] 추천 조합 수락 or 직접 키워드 2~3개 선택
[1:00] 생성 모드 선택 (Full Match / Forced Pairing)
[1:30] LLM이 아이디어 10개 생성 (제목 + 한 줄 요약)
[3:30] 마음에 드는 1~2개에 ⭐ 북마크
[4:00] Notion에 자동 저장 (북마크 + 비선택 모두 로그)
[4:30] 완료. 하루 시작.
```

### 4.3 저녁/주말 — 수렴 세션 (선택적)

```
1. 북마크된 아이디어 목록에서 1개 선택
2. Deep Report(PRD) 생성 요청
3. 비즈니스 평가 매트릭스 실행
4. Next Step 제안 확인
5. Notion 상세 페이지로 저장
6. 아이디어 상태 업데이트 (관심 → 검토 중 등)
```

## 5. Deep Report (PRD) 템플릿

선택한 아이디어에 대해 아래 구조의 상세 보고서를 생성한다:

```
1. 아이디어 요약 (엘리베이터 피치, 3문장 이내)
2. 문제 정의 (타겟의 Pain Point)
3. 솔루션 개요 (핵심 기능 3가지)
4. 타겟 사용자 페르소나 (1개)
5. 경쟁 환경 분석 (유사 서비스 2~3개, 차별점)
6. 수익 모델 (선택된 Money 키워드 기반 구체화)
7. MVP 범위 (4주 내 구현 가능한 최소 기능)
8. 핵심 리스크 (기술적/시장적 리스크 각 1개)
9. 필요 리소스 (기술 스택, 예상 개발 기간)
```

## 6. 비즈니스 평가 매트릭스 (Evaluation Matrix)

### 6.1 평가 항목 및 가중치

| 카테고리 | 가중치 | 평가 기준 |
| --- | --- | --- |
| **시장성 (Market)** | **0.30** | Problem-Solution Fit, 시장 규모 및 타겟 명확성 |
| **실행 가능성 (Build)** | **0.25** | 기술적 구현 난이도, MVP 제작 소요 기간 |
| **독창성 (Edge)** | **0.25** | Popping(톡톡 튀는) 정도, 경쟁 우위(Moat) |
| **수익성 (Money)** | **0.20** | 수익 모델의 명확성 및 지속 가능성 |

> **윤리성(Ethics):** 별도 항목으로 분리. 점수에 포함하지 않되, 치명적 윤리 리스크 발견 시 ⚠️ 플래그로 경고 표시.

### 6.2 Hallucination 완화 — 3중 출력 구조

각 평가 항목은 반드시 아래 포맷으로 출력한다:

```
시장성: 72/100
━━━━━━━━━━━━━━━━━━━━━━
근거: [왜 이 점수인지 구체적 이유 2~3문장]
반론: [이 점수가 과대/과소 평가일 수 있는 이유 1~2문장]
확인 필요: [검증하면 점수가 바뀔 수 있는 핵심 가정 1개]
```

- **점수:** 참고용 수치. 절대적 신뢰가 아닌 상대적 감각으로 활용
- **근거:** 점수의 이유. 이것이 핵심 아웃풋
- **반론:** LLM에게 자기 판단을 의심하게 하여 hallucination 가시화
- **확인 필요:** 사용자가 추후 리서치할 때의 체크리스트로 활용

### 6.3 평가 이후 — Next Step 제안

평가 완료 후, 종합 점수 구간에 따라 구체적 다음 행동을 자동 제안한다:

| 종합 점수 | 제안 액션 |
| --- | --- |
| **80+** | "경쟁사 3곳 심층 리서치", "랜딩 페이지 초안 작성", "1주 MVP 스프린트 계획 수립" |
| **60~79** | "확인 필요 항목 우선 리서치", "피벗 포인트 탐색 (약점 카테고리 재설계)" |
| **60 미만** | "핵심 가치 재정의", "키워드 조합 변경 후 재시도", "보류 처리" |

## 7. 아이디어 생명주기 관리 (Idea Lifecycle)

### 7.1 상태 관리 (Kanban)

```
신규(New) → 관심(Interested) → 검토 중(Reviewing) → 실행(Executing) / 보류(On Hold) / 폐기(Archived)
```

- **아침 발산 세션:** ⭐ 북마크된 아이디어 → `관심` 상태로 자동 이동
- **수렴 세션에서 Deep Report 생성 시:** → `검토 중`으로 전환
- **방치 감지:** 14일 이상 상태 변화 없는 아이디어에 `재검토 필요` 태그 자동 부여

### 7.2 중복 감지 메커니즘

- **V1 (최소 구현):** 아이디어 생성 시, 최근 30일 내 생성된 아이디어 제목 리스트를 프롬프트에 포함하여 "아래 아이디어는 이미 생성되었으므로 제외" 지시
- **V2 (고도화):** 아이디어 임베딩 기반 유사도 체크. 코사인 유사도 0.85 이상인 기존 아이디어 존재 시 경고 표시

### 7.3 피드백 루프 — 시스템 학습

사용자의 선택/비선택 패턴을 Notion에 로그로 축적하여 시스템 개선에 활용한다:

- **V1:** 매 세션마다 `{선택된 아이디어 ID, 비선택 아이디어 ID, 사용된 키워드, 생성 모드, 날짜}` 로그 저장
- **V2:** 축적된 로그 분석으로 사용자 선호 프로파일 생성 (선호 도메인, 선호 수익 모델 등), 추천 조합 알고리즘에 반영

## 부록 A. 프롬프트 설계 방향 (V1)

### A.1 아이디어 생성 프롬프트 — 핵심 원칙

- 키워드 조합을 "제약 조건"이 아닌 "영감의 씨앗"으로 해석하도록 지시
- 10개 아이디어는 서로 다른 접근 방식을 취하도록 강제 (예: 최소 3개는 B2C, 3개는 B2B, 나머지 자유)
- 이전 생성 아이디어 제목을 컨텍스트에 포함하여 중복 방지
- 각 아이디어는 `제목(10자 내외) + 한 줄 요약(30자 내외)` 형식 준수

### A.2 평가 프롬프트 — 핵심 원칙

- 반드시 3중 구조(근거 + 반론 + 확인 필요) 출력 강제
- "확신할 수 없는 부분은 점수를 낮추기보다 '확인 필요'에 명시하라" 지시
- 평가 시 해당 아이디어의 PRD 전문을 컨텍스트에 포함
- 점수 인플레이션 방지를 위해 "60점이 평균" 기준선 명시
