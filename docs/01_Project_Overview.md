## 1. 프로젝트 정의

**Idea Bank**는 매일 아침 짧은 시간 안에 비즈니스 아이디어를 발산하고, 이후 수렴 세션에서 실행 가능성을 검토할 수 있도록 설계한 개인용 아이디어 워크벤치다.

핵심 정의:

- **목표:** 매일 5분 안에 새로운 조합으로 아이디어를 생성하고 축적한다.
- **방식:** 고정 키워드 프레임워크와 사용자 커스텀 키워드를 결합한다.
- **철학:** 인간의 직관과 AI의 생성 능력을 결합한 `Human-in-the-Loop` 시스템을 지향한다.
- **저장소:** 아이디어, 평가, 세션 로그는 모두 Notion에 구조화해 누적한다.

> 한 줄 정의: "매일 아침 5분, 체계적 키워드 조합으로 비즈니스 아이디어를 발산하고 축적하는 개인 생산성 엔진"

### 1.1 해결하려는 문제

현재 문제는 세 가지다.

- 자유형 브레인스토밍은 매번 출발점이 달라 아이디어 품질 편차가 크다.
- 괜찮았던 아이디어도 기록과 분류가 약하면 다시 찾기 어렵다.
- 생성과 평가는 분리되어 있어, 재미있는 아이디어가 실행 가능한 아이디어로 잘 이어지지 않는다.

Idea Bank는 위 문제를 `구조적 발산 → 선택 → 평가 → 축적` 루프로 줄이는 것을 목표로 한다.

### 1.2 V1이 성공한 상태

V1은 아래 상태를 달성하면 "운영 가능"으로 본다.

- 5분 발산 세션이 주 4회 이상 무리 없이 돌아간다.
- 북마크된 아이디어가 Deep Report와 평가까지 실제로 이어진다.
- 중복 아이디어가 통제되고, 예전 아이디어를 다시 찾을 수 있다.
- 운영 중인 규칙이 다음 버전 구현의 기준 문서 역할을 한다.

## 2. 타겟 사용자와 사용 맥락

| 항목 | 결정 |
| --- | --- |
| 타겟 사용자 | 본인 (AI 엔지니어, 개인 생산성 도구) |
| 사용 패턴 | 매일 아침 5분 발산, 저녁/주말 수렴 |
| 저장소 | Notion |
| 기술 스택 | V1: Claude Project + Notion API, V2: Next.js 웹앱 |
| 운영 성격 | Solo 프로젝트, 개인 아이디어 발굴 및 검토 도구 |

주요 사용 시나리오:

- 매일 아침 추천 조합 또는 직접 선택한 키워드로 아이디어 10개를 발산한다.
- 그중 북마크한 아이디어를 저녁이나 주말에 다시 꺼내 Deep Report와 평가를 진행한다.
- 결과를 Notion에 누적하여 반복 패턴, 선호 키워드, 검토 이력을 남긴다.

## 3. 핵심 차별점

기존 대안(ChatGPT Custom GPTs, Notion AI, IdeaBuddy, Validator AI 등)과의 차별점은 아래 세 가지다.

- **체계적 키워드 조합:** 단순 자유 발상이 아니라 5개 카테고리 기반의 구조적 발산을 사용한다.
- **평가 프레임워크 내장:** 생성에서 끝나지 않고 PRD와 평가까지 연결해 실행 가능성 판단을 돕는다.
- **축적과 학습:** 세션 로그와 선택/비선택 결과를 저장해 이후 추천과 회고에 활용한다.

## 4. 핵심 기능 요약

| 기능 | 설명 |
| --- | --- |
| 키워드 매트릭스 | `Who / Domain / Tech / Value / Money` 5개 축으로 아이디어의 영감 입력값을 구조화 |
| 생성 모드 | Full Match, Forced Pairing, Serendipity 3가지 모드 제공 |
| 추천 조합 | 최근 사용 패턴과 다양성 기준을 반영해 매일 추천 조합 제시 |
| Deep Report | 선택한 아이디어를 PRD 형태로 확장 정리 |
| 평가 매트릭스 | 시장성, 실행 가능성, 독창성, 수익성을 가중치 기반으로 평가 |
| Lifecycle 관리 | 북마크, 상태 전환, 중복 감지, 방치 감지, 세션 로그를 통한 운영 관리 |
| Notion 축적 | Ideas, Evaluations, Session Logs, Keywords 테이블에 결과 누적 |

## 6. 단계별 로드맵

### 6.1 V1 (MVP)

목표는 **2주 내 운영 시작**이다.

핵심 범위:

- Claude Project 기반 아이디어 생성
- 고정 키워드 90+ 정리 및 프롬프트 반영
- 커스텀 키워드 수동 추가/삭제
- Full Match / Forced Pairing 생성 모드
- 오늘의 추천 조합
- Deep Report 생성
- 평가 매트릭스 및 Next Step 제안
- Notion 연동
- 최근 30일 제목 기반 중복 방지
- 아이디어 Kanban 상태 관리
- 14일 방치 감지

### 6.2 V2

V1 안정화 이후 다음 범위로 확장한다.

- Next.js 웹앱 마이그레이션
- 다이내믹 키워드 자동 생성 파이프라인
- 임베딩 기반 중복 감지
- 사용자 선호 프로파일 생성
- 추천 알고리즘 고도화
- 피드백 루프 대시보드
- 키워드 사용 통계 시각화

### 6.3 V2 착수 게이트

V2는 "하고 싶을 때"가 아니라, 아래 조건이 충족될 때만 착수한다.

| 게이트 | 기준 |
| --- | --- |
| 루틴 정착 | 최근 4주 기준 주간 발산 세션 4회 이상 |
| 수렴 전환 | 최근 4주 기준 월간 Deep Report 4회 이상 |
| 품질 안정 | 중복 아이디어 비율 15% 미만 |
| 실행 연결 | 최근 4주 기준 최소 1개 아이디어가 `실행` 또는 외부 검증 단계로 이동 |

상세 운영 수치는 [04_Operations_Handbook.md](./04_Operations_Handbook.md)를 기준으로 관리한다.

## 7. 제약 조건 및 운영 원칙

- **개인용 우선:** 멀티 유저나 협업 기능보다 개인 루틴 정착을 먼저 검증한다.
- **Notion 중심 운영:** V1에서 Notion을 단일 저장소로 사용한다.
- **세션 분리:** 발산과 수렴을 같은 세션에 몰아넣지 않는다.
- **비용 제한:** 월간 API 예산 상한과 세션 횟수 제한을 둔다.
- **검증 후 확장:** 다이내믹 키워드, 임베딩, 개인화 추천은 V1 안정화 후 진행한다.
