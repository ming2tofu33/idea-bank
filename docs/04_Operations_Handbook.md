## 1. 운영 개요

Idea Bank 운영은 발산과 수렴을 분리하는 것이 핵심이다.

운영 원칙:

- 아침에는 빠른 발산만 수행한다.
- 저녁이나 주말에는 선택한 아이디어만 깊게 검토한다.
- 전체 워크플로우를 한 세션에 몰아넣지 않는다.
- 월간 비용과 주간 검토량에 상한을 둔다.

---

## 2. 세션 운영

### 2.1 아침 5분 발산 세션

```text
[0:00] 앱 열기 → 오늘의 추천 조합 3세트 표시
[0:30] 추천 조합 수락 또는 직접 키워드 2~3개 선택
[1:00] 생성 모드 선택 (Full Match / Forced Pairing)
[1:30] LLM이 아이디어 10개 생성
[3:30] 마음에 드는 1~2개 북마크
[4:00] Notion에 자동 저장 (북마크 + 비선택 모두 기록)
[4:30] 종료
```

운영 fallback:

- 발산 세션이 주 3회 이상 5분을 넘기면 다음 주부터 기본 검토 대상을 `상위 3개 후보 + 나머지 펼치기` 방식으로 줄인다.
- 북마크 선택이 반복해서 지연되면 생성 수를 유지하되 요약 길이를 더 짧게 제한한다.

### 2.2 저녁/주말 수렴 세션

```text
1. 북마크된 아이디어 중 1개 선택
2. Deep Report(PRD) 생성
3. 비즈니스 평가 매트릭스 실행
4. Next Step 확인
5. Notion 상세 페이지 저장
6. 아이디어 상태 업데이트
```

---

## 3. 운영 리듬

| 주기 | 운영 내용 |
| --- | --- |
| 매일 | 발산 세션 1회 수행, 필요 시 커스텀 키워드 수동 추가 |
| 주 1~2회 | Deep Report 생성 및 평가 진행 |
| 주간 | 북마크 아이디어 검토, 상태 전환, 보류 여부 결정 |
| 분기 1회 | 전체 키워드 풀 리뷰 및 정리 |

추가 원칙:

- 추가 발산 세션이 필요하면 허용하되 비용 경고를 표시한다.
- 14일 이상 상태가 멈춘 아이디어는 재검토 대상으로 올린다.

## 4. 비용 관리

### 4.1 운영 모드 구분

| 운영 모드 | 설명 | 주 비용 포인트 |
| --- | --- | --- |
| V1 Claude Project 모드 | Claude Pro 중심 운영, 수동 또는 저비용 자동화 연결 | 구독 비용, Notion/n8n/Make 운영 오버헤드 |
| V2 API 모드 | Next.js 앱 + Anthropic API 직접 호출 | 토큰 비용, 저장/관측 파이프라인 비용 |

### 4.2 API 기준 비용 추정

| 항목 | 빈도 | 예상 토큰 | 월간 비용(추정) |
| --- | --- | --- | --- |
| 아침 발산 세션 | 매일 | 약 2K input + 2K output | 약 $3~5 |
| Deep Report | 주 2회 | 약 3K input + 4K output | 약 $2~4 |
| 비즈니스 평가 | 주 2회 | 약 3K input + 3K output | 약 $2~3 |
| 월간 합계 |  |  | 약 $7~12 |

운영 메모:

- 위 표는 **V2 API 모드 기준** 추정치다.
- V1은 API 비용보다 운영 마찰과 자동화 연결 안정성이 더 큰 관리 대상이다.
- Notion, n8n/Make 사용 여부는 별도 운영 메모로 월 1회 기록한다.

### 4.3 비용 상한

- 월간 API 예산 상한: `$30`
- Deep Report 생성 제한: `주 5회`
- 일일 발산 세션 기준: `1회`

### 6.3 V1 Done 기준

아래 조건을 모두 만족하면 V1을 "운영 가능"으로 판정한다.

- 발산 세션이 저장까지 끊기지 않고 5회 연속 수행된다.
- Deep Report와 평가가 같은 아이디어에 대해 최소 3회 이상 완료된다.
- 중복 경고, 저장 실패, 상태 전이 규칙이 문서와 실제 운영에서 충돌하지 않는다.
- 운영 메모에 다음 수정 우선순위를 남길 수 있을 정도로 사용 패턴이 관찰된다.
