## 1. 키워드 시스템 개요

Idea Bank의 아이디어 생성은 5개 축의 키워드 조합을 중심으로 돌아간다.

핵심 원칙:

- 고정 키워드 풀을 기본 프레임워크로 사용한다.
- 사용자가 커스텀 키워드를 추가해 관심사를 반영할 수 있다.
- V2에서는 동적 키워드를 자동 수집해 확장한다.
- 추천 로직은 다양성과 최근 사용 패턴을 함께 고려한다.

카테고리 축:

1. Who
2. Domain
3. Tech
4. Value
5. Money

### 2.2 키워드 풀 관리 원칙

- 분기 1회 전체 키워드 풀 리뷰
- 카테고리별 추상도 균형 유지
- 의미적으로 겹치는 키워드는 통합
- 3개월간 거의 쓰지 않은 키워드는 정리 후보로 표시

### 2.3 추상도 정규화 규칙

카테고리별 추상도는 아래 수준을 기본으로 맞춘다.

- **Who:** 사용자군 또는 생활/업무 맥락 수준
- **Domain:** 산업 vertical 수준
- **Tech:** 구현 접근 방식 또는 핵심 기술 수준
- **Value:** 사용자가 체감하는 효용 수준
- **Money:** 수익화 메커니즘 수준

분기 리뷰 때 위 수준에서 너무 넓거나 좁은 항목은 sibling 수준으로 재작성하거나 별도 후보군으로 분리한다.

## 3. 커스텀 키워드와 다이내믹 키워드

### 3.1 커스텀 키워드

- 사용자가 직접 키워드를 추가/삭제한다.
- V1에서는 수동 운영을 기본으로 한다.
- V2 자동화와 연결할 수 있도록 데이터 구조는 미리 통일한다.

필수 데이터 구조:

```json
{
  "keyword": "AI 코파일럿",
  "category": "Tech",
  "source": "custom",
  "added_date": "2025-03-07",
  "used_count": 0,
  "last_used": null
}
```

### 3.2 다이내믹 키워드

V2에서만 도입한다.

- 최신 IT 뉴스와 SNS 트렌드를 분석해 신규 키워드를 자동 생성
- 뉴스 소스 선정, 수집 파이프라인, 키워드 추출, 중복 제거, 스케줄링 필요
- V1 안정화 이전에는 도입하지 않는다

## 4. 아이디어 생성 모드

### 4.1 Full Match

사용자가 선택한 모든 키워드를 반드시 반영해 아이디어를 생성한다.

적합한 상황:

- 특정 제약 조건을 모두 반영한 아이디어를 보고 싶을 때
- 문제 정의가 비교적 선명할 때

### 4.2 Forced Pairing

`고정 키워드 2개 + 커스텀 키워드 1개` 조합을 우선으로 강제해 의외성 있는 아이디어를 끌어낸다.

적합한 상황:

- 평소 떠올리지 않는 조합을 보고 싶을 때
- 의도적 충돌이나 신선한 발상을 유도하고 싶을 때

Fallback 규칙:

- 커스텀 키워드가 없으면 `고정 키워드 3개` 조합으로 대체한다.
- 커스텀 키워드가 2개 이상이면 최근 14일간 사용 빈도가 가장 낮은 항목을 우선 사용한다.
- 최근 30일 내 같은 조합이 이미 쓰였다면 다음 후보 조합으로 넘어간다.

### 4.3 Serendipity

결정 피로를 줄이기 위한 추천 조합 중심 모드다.

핵심 동작:

- 이전에 사용하지 않은 조합을 우선 제안
- 최근 자주 선택한 키워드와 일부러 다른 조합을 섞음
- 추천 조합 수락 후 곧바로 생성으로 이어짐

추천 로직:

`사용 빈도 낮은 키워드 가중 + 카테고리 간 다양성 확보`

기본 점수식:

`serendipity_score = novelty(0.5) + diversity(0.3) + recency_penalty(0.2)`

판정 규칙:

- **novelty:** 최근 30일 내 미사용 조합일수록 높게 부여
- **diversity:** 최근 14일간 자주 등장하지 않은 카테고리 조합일수록 높게 부여
- **recency_penalty:** 직전 7일간 동일 키워드 재등장 비율이 높을수록 감점

추천 후보 필터:

- 최근 7일간 동일 조합 재노출 금지
- 동일 카테고리 내부 의미 중복 키워드 동시 사용 금지
- 동적 키워드는 검증 전 최대 1개까지만 포함

## 5. 아이디어 생명주기

### 5.1 상태 관리

```text
신규(New) → 관심(Interested) → 검토 중(Reviewing) → 실행(Executing) / 보류(On Hold) / 폐기(Archived)
```

상태 전이 규칙:

- 아침 발산 세션에서 북마크한 아이디어는 `관심`으로 이동
- 수렴 세션에서 Deep Report를 생성하면 `검토 중`으로 전환
- 14일 이상 상태 변화가 없으면 `재검토 필요` 태그를 부여

### 5.2 중복 감지

- **V1:** 최근 30일 내 생성된 제목을 프롬프트에 넣어 유사 제목 재생성을 줄인다.
- **V2:** 임베딩 기반 유사도 검사를 추가하고 코사인 유사도 0.85 이상일 때 경고한다.

### 5.3 피드백 루프

선택/비선택 패턴은 세션 로그로 누적한다.

- **V1:** 선택된 아이디어, 비선택 아이디어, 사용 키워드, 생성 모드, 날짜 기록
- **V2:** 누적 로그를 기반으로 선호 도메인, 선호 수익 모델 등 개인 프로파일 생성

### 5.4 평가 결과에 따른 상태 전이

평가 이후 상태는 아래 규칙으로 정한다.

| 조건 | 상태 전이 |
| --- | --- |
| 종합 점수 80 이상 | `검토 중` 유지, 실행 후보 태그 부여 |
| 종합 점수 60~79 | `관심` 또는 `검토 중` 유지, `추가 리서치 필요` 태그 부여 |
| 종합 점수 60 미만 | 기본값 `보류` |
| 중복 경고 발생 | 새 아이디어 생성 대신 기존 아이디어와 병합 검토 |
| 보류 아이디어 재검토 | 수동 승인 시 `관심`으로 복귀 |
